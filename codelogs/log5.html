Using images in HTML vs. CSS backgrounds

Images can be inserted directly into the document flow using the HTML <img> element or applied as decorative or layout backgrounds using CSS background-image. Each approach has different code patterns and use cases.

Differences (code and behavior)

  • <img> is semantic content: browsers, search engines and assistive tech see it as part of the page. It supports alt text, srcset/sizes for responsive images, and intrinsic dimensions.
  • background-image is presentation only: it belongs to the element's box and is not announced by screen readers. Best for decorative images or layout effects (e.g., hero backgrounds, patterned tiles).
  • Layout: <img> participates in flow and affects document size; background images do not change element dimensions (except as styled) and can be clipped or repeated.

Other CSS properties that affect background images

  • background-size — controls scaling (e.g., cover, contain, or explicit sizes like 50% 100%).
  • background-position — controls the image's alignment inside the box (e.g., center, top right, or numeric offsets).
  • background-repeat — controls tiling (repeat, no-repeat, repeat-x, space).
  • background-attachment — can fix a background with respect to the viewport (fixed) or scroll with the content (scroll).

Image sizing and fitting

For replaced elements like <img> the object-fit property controls how the image fills its content box (similar to background-size for CSS backgrounds).

Dimensions vs. resolution (one sentence)

Dimensions are the pixel width and height of a bitmap (e.g., 1200×800 px); resolution describes pixel density (pixels per inch or device pixels per CSS pixel), which affects sharpness on different screens.

Extra credit — image formats: similarities, differences, and uses

FormatNotes & best uses
GIF Indexed 256-color palette, supports simple animation and lossless compression for small graphics; good for low-color icons and simple animations but poor for photos.
PNG Lossless, supports full transparency (alpha), good for screenshots, icons, and images needing crisp edges; larger file sizes for photos compared with lossy formats.
JPG / JPEG Lossy compression optimized for photos; much smaller files for complex imagery but not ideal for sharp text or transparency; quality vs. size trade-off via compression level.
WebP Modern format supporting both lossy and lossless compression plus alpha and animation; often smaller than JPEG/PNG — good for web delivery when supported.
SVG Vector format (XML); infinitely scalable without quality loss, small for icons/illustrations, scriptable and styleable with CSS; not suitable for complex photographs.

Summary: choose SVG for scalable graphics and icons, JPEG/WebP for photos, PNG for transparency and lossless needs, GIF for simple animations or tiny palettes; prefer WebP where browser support and tooling allow for best compression.
